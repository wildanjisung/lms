Problem map and recommended fixes.

This page is the short correction summary. Use the compare view for the detailed side-by-side discussion, then keep this table as the quick reference.

Category Page Issue Observed problem Suggested fix
Labels and instructions Home Search field The search box relies on placeholder text only, so the purpose disappears as soon as the user types. Add a visible label and keep the helper text outside the placeholder.
Keyboard / operable Home Featured card The feature card is only clickable with a pointer, so keyboard users cannot activate it. Use a real link or button and keep a visible focus state.
Contrast Program Status text The status text is too light against the background, so it is harder to read. Use sufficient contrast for text and badges.
Semantics Program Schedule grid The schedule is built from generic cards, so the relationships between time and session are not explicit. Use a semantic table or another structure with clear headers and rows.
Labels and instructions Tickets Email field The email field uses placeholder text only, which makes the instruction brittle and easy to miss. Add a visible label and a short support hint below the field.
Error identification Tickets Validation message The error copy is vague and leans on color alone instead of explaining the exact problem. Name the failing field and say what format or value is expected.
Media / text alternatives Venue Media block The media card lacks captions, transcript access, and meaningful alternative text for the event images. Provide alt text, captions, and a transcript link beside the player.
