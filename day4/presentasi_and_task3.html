Issue map

Name the barrier, the impact, and the fix.

This slide is the main discussion surface. Keep the explanation direct and concrete.

Labels and instructions Search field on Home

The search box relies on placeholder text only, so the purpose disappears as soon as the user types.

Fix: Add a visible label and keep the helper text outside the placeholder.
Keyboard / operable Featured card on Home

The feature card is only clickable with a pointer, so keyboard users cannot activate it.

Fix: Use a real link or button and keep a visible focus state.
Contrast Status text on Program

The status text is too light against the background, so it is harder to read.

Fix: Use sufficient contrast for text and badges.
Semantics Schedule grid on Program

The schedule is built from generic cards, so the relationships between time and session are not explicit.

Fix: Use a semantic table or another structure with clear headers and rows.
Labels and instructions Email field on Tickets

The email field uses placeholder text only, which makes the instruction brittle and easy to miss.

Fix: Add a visible label and a short support hint below the field.
Error identification Validation message on Tickets

The error copy is vague and leans on color alone instead of explaining the exact problem.

Fix: Name the failing field and say what format or value is expected.
Media / text alternatives Media block on Venue

The media card lacks captions, transcript access, and meaningful alternative text for the event images.

Fix: Provide alt text, captions, and a transcript link beside the player.
For each issue, say what is broken, who it affects, and what a clean fix looks like.
