Accessibility

Accessibility is product quality.

An interface is not finished if people cannot perceive, understand, or operate it.

Keyboard users

Every control must be reachable without a mouse.

Screen reader users

Controls need names, roles, labels, and structure.

Users with visual barriers

Text needs readable contrast and non-color cues.

Define accessibility in practical product terms. The goal is simple: more people can complete the task with less friction.

Barrier patterns

Common barriers repeat.

Most beginner findings fit into three practical groups.

Language

Unlabeled fields, vague links, unclear buttons.

Visibility

Low contrast, color-only states, icons with no meaning.

Operation

Keyboard skips, focus traps, weak guidance.

Use categories to keep feedback structured. This prevents learners from writing random comments.

Task assignment

Task Assignment

Original wording from the course plan.

Task 1: Barriers in a Web interface identify
The Participants consider a simple Web interface from Everyday perspective, to Example a Kontaktformular, a Login-Area, a Course registration, a Shopping cart or a Navigationsseite. The Task is without technical Prior knowledge solvable, because They from Nutzerperspektive begins.
Assignment:
1. Describe They, which Elements on the Interface visible are.
2. Check They, whether the Elements understandably beschriftet are.
3. Consider They, whether the Page without Mouse usable would be.
4. Evaluate They, whether Farben and Contrasts sufficient clear seem.
5. Check They, whether Error messages understandably would be.
6. Name They Elements, the for Screenreader problematisch are could.
7. Formulate They at least five possible Barriers.
8. Describe They to each Barriere a simple Verbesserungsidee.
Exemplary Barriers: Button without clear Designation, Formularfeld without Label, Error message only in Rot, to smaller Kontrast, relevant factor only per Mouse reachable, Linktext „here klicken“, unclear Order when relevant factor, missing Description when Symbolen.
Objective the Task: The Participants identify, that digital accessibility already when simple relevant factor begins. They develop a first Understanding for this, how Design, Structure and Usability zusammenwirken.
Expected Result: A Barrierenliste with Improvement proposals for Structure, understandability, Usability and visual Wahrnehmbarkeit.
This slide preserves the assignment wording as written in the course plan. Use it when introducing the task requirements.
